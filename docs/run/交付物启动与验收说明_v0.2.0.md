## 2. 启动步骤

### 2.1 可选：启动后端联调环境

若需要验证调试页中的录入/查询能力，可先启动 v0.1.0 后端环境:

```bash
cd /Users/fangyongchao/Projects/on_the_way
cp infra/docker/.env.example infra/docker/.env
docker compose -f infra/docker/docker-compose.yml --env-file infra/docker/.env up -d
```

### 2.2 启动 iOS 客户端

终端 A:

```bash
cd /Users/fangyongchao/Projects/on_the_way/apps/ios
npm install
npm run pods
npm start
```

终端 B:

```bash
cd /Users/fangyongchao/Projects/on_the_way/apps/ios
npm run ios -- --simulator "iPhone 17"
```

## 3. UI 验收清单（v0.2.0）

### 3.1 首页结构

- 首屏显示 `On The Way` 与副标题 `今日概览`。
- 显示当天日期。
- 显示 8 个主题导航标签（默认选中 `健康`）。
- 显示 2 列 x 4 行主题卡片，每张卡有主题名、说明、状态标签 `即将开放`。
- 显示页脚文案:
  - `v0.2.0 当前为界面设计版本，更多功能将分阶段开放。`

### 3.2 交互行为

- 点击任一主题标签:
  - 标签进入选中态
  - 对应卡片同步高亮
  - 显示统一提示（即将开放）
- 点击任一卡片:
  - 有按压反馈
  - 显示统一提示（即将开放）
  - 不触发新增业务逻辑与新增 API 调用

### 3.3 文案约束

- 客户端可见文案不出现 `系统` 字样。

### 3.4 回归验证（v0.1.0 调试入口）

- 在首页长按品牌区可进入调试页。
- 调试页内可切换:
  - `手工录入`
  - `结果查看`
- 若后端已启动，原有 `manual-signal` / `daily-summary` 联调能力可继续使用。
