# On The Way 产品文档 v0.2.0

版本: `v0.2.0`  
日期: `2026-02-26`  
状态: `已验收（已完成）`  
面向角色: 产品、UI/UX 设计、iOS 客户端研发、测试

---

## 1. 版本定位与范围

### 1.1 v0.2.0 目标
- 在 `v0.1.0` 可运行基线上，完成 iOS 客户端首页的 UI/UX 重构。
- 提供面向未来 8 个主题域的统一导航入口（首页导航栏）。
- 建立可复用的 MUJI（无印良品）风格视觉规范，作为后续页面设计基线。

### 1.2 基线输入（来自 v0.1.0）
- 数据平台链路与 API 已可用（`ingest/query` 已打通）。
- iOS 现有页面为 POC 录入/查询页面，视觉为工程化默认样式。
- 本版本不改动服务端、数据模型、接口协议。

### 1.3 硬约束
- 不做功能开发: 不新增后端 API、不新增数据模型、不新增业务流程。
- 仅做 iOS 客户端 UI/UX 设计与前端界面实现。
- 首页必须包含后续 8 个主题入口导航栏。
- 视觉风格必须遵循 MUJI 风格（克制、自然、留白、低饱和中性色）。
- 客户端展示文案中禁止出现“系统”字样（例如“个人经营系统”“健康系统”）。

### 1.4 非目标
- 不在 v0.2.0 实现 8 个系统的业务页面与数据读写。
- 不在 v0.2.0 做登录鉴权、权限系统、消息系统。
- 不在 v0.2.0 改造现有数据链路（Raw/Canonical/Domain/Mart）。

## 3. 首页信息架构（IA）

### 3.1 首页模块结构
1. 顶部品牌区
- App 名称: `On The Way`
- 副标题: `今日概览`（固定文案）
- 当天日期（只读展示）

2. 8 主题导航栏（必须项）
- 位置: 顶部品牌区下方
- 形式: 横向滚动导航栏（8 个主题标签）
- 默认选中: `健康`

3. 主题导航卡片区
- 形式: `2 列 x 4 行` 卡片栅格
- 每张卡片包含: 主题名 + 一句话说明 + 状态标签（`即将开放`）

4. 页脚说明区
- 固定文案: `v0.2.0 当前为界面设计版本，更多功能将分阶段开放。`

### 3.2 八大主题导航定义（内部对应八大经营域）

## 4. MUJI 风格设计规范

### 4.1 设计关键词
- 自然
- 克制
- 安静
- 清晰
- 高可读性

### 4.2 颜色令牌
```text
color.bg.canvas      = #F6F3ED
color.bg.surface     = #EFE8DC
color.bg.card        = #FBF9F4
color.text.primary   = #2F2A24
color.text.secondary = #6B645A
color.border.default = #D8D0C4
color.accent.soft    = #8A7E70
color.state.disabled = #B8B1A7
```

使用规则:
- 页面主背景使用 `canvas`。
- 卡片背景使用 `card`，边框统一 `border.default`。
- 仅允许 `accent.soft` 用于选中态或轻强调，不使用高饱和强调色。

### 4.3 字体与排版
- 字体族: iOS 系统字体（`SF Pro` / `PingFang SC`）
- 标题（H1）: 28/34, SemiBold
- 模块标题（H2）: 20/26, Medium
- 正文: 16/24, Regular
- 辅助文案: 13/18, Regular

### 4.4 间距与圆角
- 栅格基线: `4pt`
- 常用间距: `8/12/16/20/24`
- 卡片圆角: `12`
- 导航标签圆角: `999`（胶囊）

### 4.5 图标与插图
- 图标风格: 单色线性图标，线宽统一。
- 图标资产: 本版本必须交付 8 个主题的正式图标，不使用占位图标。
- 禁止使用拟物高光、荧光色、重渐变背景。

### 4.6 文案约束（客户端展示层）
- 禁用词: `系统`、`个人经营系统`、`健康系统`、`时间系统` 等同类文案。
- 允许词: `首页`、`主题`、`分类`、`健康`、`时间`、`收入`、`财务`、`能力`、`关系`、`生活`、`保障`。

## 5. 交互与动效规范

### 5.1 导航栏交互
- 点击标签后，标签进入选中态（背景 `surface`，文字 `text.primary`）。
- 内容区同步高亮对应主题卡片。

### 5.2 卡片交互
- `default`: 常规样式。
- `pressed`: 透明度 0.9 + 轻微缩放（0.98）。
- `disabled`/`coming soon`: 仍可点击显示占位提示，但不跳转。

### 5.3 动效参数
- 时长: `160ms-220ms`
- 曲线: `ease-out`
- 原则: 仅做状态反馈，不做复杂转场。

## 7. 可访问性与质量要求

### 7.1 可访问性
- 点击热区不小于 `44x44 pt`。
- 正文与背景对比度不低于 WCAG AA（普通文本建议 `>= 4.5:1`）。
- 支持系统字体放大后的基本可读与不重叠。

### 7.2 质量要求
- 不得出现布局溢出、文字截断（主题名除外，最多单行省略）。
- 首屏渲染无明显卡顿。
- 保证 iOS 模拟器与真机显示一致性。

## 10. 已确认产品决策

1. 首页不保留 `v0.1.0` 的“手工录入/结果查看”可见入口。  
说明: 仅保留开发调试入口，不在主导航和首页主区域展示。

2. 八大主题采用统一正式图标资产。  
说明: 本版本交付正式图标，不使用占位图标。

4. 整个客户端可见文案禁止出现“系统”相关字样。  
说明: 使用“主题/分类/首页”等中性表达。
